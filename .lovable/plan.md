

# Authentication, Cloud Sync & Fullscreen Columns

## 1. Enable Lovable Cloud (Supabase)
- Enable Supabase via Lovable Cloud for auth and database

## 2. Database Schema
- **folders** table: `id (uuid PK)`, `user_id (uuid FK auth.users)`, `name (text)`, `created_at (timestamptz)`
- **songs** table: `id (uuid PK)`, `user_id (uuid FK auth.users)`, `folder_id (uuid FK folders)`, `title (text)`, `content (text)`, `created_at (timestamptz)`
- RLS on both tables: users can only CRUD their own rows

## 3. Auth Pages
- Create `src/pages/Auth.tsx` — simple email/password login + signup form with toggle between modes
- Add route `/auth` in `App.tsx`
- Create auth context/hook (`useAuth`) using `supabase.auth.onAuthStateChange`
- Protect the main Index page — redirect to `/auth` if not logged in
- Add logout button in sidebar

## 4. Migrate Data Layer to Supabase
- Rewrite `useChordBook` to use Supabase queries (via `@tanstack/react-query` for caching) instead of localStorage
- All CRUD operations become Supabase inserts/updates/deletes
- On first login, if user has localStorage data, offer to import it (optional migration)

## 5. Fullscreen Column View
Based on the reference image — a distraction-free fullscreen mode with adjustable columns:

- Add a "Fullscreen" button in the `SongViewer` title bar
- When activated, use the Fullscreen API (`document.documentElement.requestFullscreen()`) to go true fullscreen
- Render song content in CSS multi-column layout (`column-count`) with a slider to adjust columns (1–4)
- Show a minimal top bar with song title, column slider, and "Exit fullscreen" button
- Auto-scroll still works in fullscreen mode
- Listen for `fullscreenchange` event to sync state if user exits via Escape
